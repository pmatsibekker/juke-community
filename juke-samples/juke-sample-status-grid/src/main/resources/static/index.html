Cross-session status grid — @Juke per-cookie replay

One recording, many concurrent replay sessions. Each worker gets its own JUKE_SESSION_ID cookie and advances through the track at its own pace; GET /service/sessions reports every session's lastCall and percentComplete — the data behind this live grid.

Drive it

Record once, then spawn workers. Each spawn starts a fresh cookie session and replays a random slice of the track, so workers land at different positions. The grid reads server-side session state, so it shows all workers — not just the one whose cookie is active.
